# 일기토 전투 규칙 명세서 (v1)

> 🗄️ **[STALE / 참고용]** 이 문서의 **4:4 턴제 일기토는 게임에서 제거됨**(2026-07 총력전 전환). 현재 일기토는 `total.html`의 **실시간 무장 결투 + 시네마틱 컷씬**으로 재설계됨(HANDOVER §4-2 일기토 참고). 이 문서는 `engine.js`가 유틸(`typeMult`/`maxHpOf`)로만 남아 참고용. 스탯/병종상성 수치의 역사적 근거로만 읽을 것.

> **이 문서의 목적**: 「대장정」의 핵심인 **4:4 턴제 일기토**를 코드로 바로 구현할 수 있게, 스탯·공식·규칙·예시 수치를 빠짐없이 정의한다.
> **읽는 사람**: ① 구현 담당(또는 Claude Code) ② 게임을 잘 모르는 기획/사업 담당.
> 게임을 모르는 분을 위해 각 규칙 끝에 **〔왜 이렇게?〕** 주석을 달았다.

> **이 버전의 범위**: 일기토 전투 **하나**만. 영지전·민심·맵·가챠·멀티는 이 전투가 "재밌다"가 확인된 뒤 바깥에 얹는다. 단, 이 전투의 **결과 출력**(맨 아래)이 나중에 사기·민심 시스템의 입력이 되도록 미리 설계해 둔다.
> **플랫폼 전제**: 웹(반응형, PC+모바일 브라우저) / 턴제 / 싱글플레이 / 서버 없음. 같은 입력이면 같은 결과(시드 기반 난수)라 테스트가 쉽다.

## 0. 한눈에 보는 전투

- 양쪽이 장수 **4명**씩 데리고 싸운다. (1:1도 같은 엔진, 인원만 1명)
- **라운드제**: 매 라운드, 살아있는 장수들이 **민첩 순서**로 한 번씩 행동한다.
- 행동은 5가지: **공격 / 방어 / 스킬 / 책략 / 사기고양**.
- 데미지는 **무력(공격) vs 통솔(방어)** + **병종 상성** + **팀 사기**로 결정된다.
- **팀 사기**가 0이 되면 **붕괴**(행동 불가 확률↑) → 이게 "소수가 대군을 무너뜨린다"의 엔진.
- 한쪽 4명이 전부 쓰러지면 끝. 결과를 바깥 게임(사기·민심)에 넘긴다.

## 1. 장수 스탯 (General Stats)

장수 한 명은 다음 값을 가진다. 4대 능력치는 모두 **1~100** 스케일.

| 키 | 이름 | 범위 | 전투에서 하는 일 |
|---|---|---|---|
| `might` | 무력(武) | 1~100 | **공격 데미지**를 키운다 |
| `command` | 통솔(統) | 1~100 | **방어력 + 최대 체력**을 키운다 |
| `intellect` | 지력(智) | 1~100 | **책략 성공률·효과**, 책략 저항 |
| `agility` | 민첩(敏) | 1~100 | **행동 순서**(빠를수록 먼저) |
| `troop` | 병종 | 보병/창병/기병/궁병 | **상성**과 기본 체력 결정 |
| `skill` | 스킬 | 아래 4종 중 1 | 장수의 시그니처 액티브 |
| `stratagem` | 책략 | 아래 2종 중 1 또는 없음 | 지력형 장수만 보유 |

> 〔왜 이렇게?〕 능력치 4개에 **각자 다른 역할**을 줬다. 무력=공격, 통솔=방어/체력, 지력=책략, 민첩=순서. 한 능력치가 여러 일을 겸하면 밸런스 잡기가 지옥이 되고, 플레이어도 "이 장수가 왜 센지" 직관이 안 온다. 1~100 스케일은 코에이 삼국지와 같아서 장수 데이터를 만들 때 감을 잡기 쉽다.

### 1.1 병종별 기본 체력 (병력)

```
보병 base HP = 1100   // 무난한 만능형, 잘 안 죽음
창병 base HP = 1000
기병 base HP = 850    // 잘 때리지만 잘 죽음(로스터에서 무력을 높게 줌)
궁병 base HP = 750    // 가장 물렁
```

**최대 체력 공식:**
```
maxHp = 병종_base_HP × (1 + command / 200)
```
- 예) 통솔 60 보병 → 1100 × (1 + 60/200) = 1100 × 1.30 = **1430**

> 〔왜 이렇게?〕 통솔 100이면 체력 +50%. 통솔이 "잘 버티는 지휘관"을 뜻하게 했다. 병종별 base는 "기병=고화력 유리몸 / 보병=탱커" 같은 성격을 만든다.

## 2. 병종 상성 (Counter Triangle)

3종 삼각형 + 보병(중립).

```
창병  →(유리)  기병  →(유리)  궁병  →(유리)  창병   (순환)
보병  ↔ 모든 병종에게 상성 보통(중립)
같은 병종끼리 → 보통
```

**상성 배수(`typeMult`):**
```
유리 = 1.3
보통 = 1.0
불리 = 0.8
```

| 공격자 \ 방어자 | 보병 | 창병 | 기병 | 궁병 |
|---|---|---|---|---|
| **보병** | 1.0 | 1.0 | 1.0 | 1.0 |
| **창병** | 1.0 | 1.0 | **1.3** | 0.8 |
| **기병** | 1.0 | 0.8 | 1.0 | **1.3** |
| **궁병** | 1.0 | **1.3** | 0.8 | 1.0 |

> 〔왜 이렇게?〕 1.3/0.8은 "유의미하지만 절대적이진 않은" 차이다. 상성으로 역전은 가능하되, 상성만으로 자동승리는 안 된다(자료 11p의 "과금=자동승리 방지"와 같은 철학). 보병을 중립으로 둬서 "안전한 만능 픽"을 만들면 초보가 덱 짜기 쉬워진다.

## 3. 팀 사기 (Team Morale) — 이 게임의 심장

각 팀(아군/적군)은 **사기 게이지 0~100**을 가진다. **시작값 50.**

### 3.1 사기가 데미지에 미치는 영향
```
moraleMult = 0.8 + (teamMorale / 100) × 0.4
```
- 사기 0 → ×0.8 / 사기 50 → ×1.0 / 사기 100 → ×1.2

### 3.2 사기 변동표

| 사건 | 그 팀 사기 | 상대 팀 사기 |
|---|---|---|
| 적 장수 처치 (kill) | +15 | −15 |
| 치명타 적중 | +5 | −5 |
| `방어` 행동 | +3 | — |
| `사기고양` 행동 | +20 | — |
| `교란`(책략) 성공 | — | −15 |
| 매 라운드 종료 | 50쪽으로 ±2 수렴 | 50쪽으로 ±2 수렴 |

> 〔왜 이렇게?〕 매 라운드 ±2 수렴은 사기가 한쪽으로 영원히 고정되는 걸 막는 안전장치다(단, 처치·고양 같은 큰 사건이 이걸 압도하게 설계). 처치 시 ±15가 양쪽에 동시에 들어가는 게 핵심 — 한 번 무너지기 시작하면 가속된다.

### 3.3 붕괴 (Rout)
팀 사기가 **0**에 닿으면 그 팀은 **붕괴 상태**:
- 그 팀 모든 장수: 매 턴 시작 시 **40% 확률로 "동요"** → 그 턴 행동 불가(스킵)
- 추가로 **공격 데미지 ×0.8**(위축)
- 팀 사기가 다시 **20 이상**으로 오르면 붕괴 해제

> 〔왜 이렇게?〕 이게 "사기 꺾이면 대군도 흔들린다"의 실제 구현이다. 회복 가능(`사기고양`·처치로)하게 만들어서 "역전 이벤트"가 살아난다. 즉 붕괴는 사형선고가 아니라 위기.

## 4. 행동 순서 (Turn Order / Initiative)

매 라운드 시작 시, 살아있고 동요 안 한 장수 전원에 대해:
```
initiative = agility + statusMod + rand(0, 10)
```
- 내림차순 정렬. 동점이면 통솔 높은 쪽 우선, 그래도 같으면 무작위.
- `statusMod`: 분기 +5 / 위축(붕괴) −5 등 상태 보정.
- **매 라운드 다시 계산**한다(상태가 라운드마다 바뀌므로).

> 〔왜 이렇게?〕 자료 5p의 "민첩으로 턴 순서 결정"을 그대로 반영. rand(0,10)은 동민첩 장수끼리 순서를 살짝 흔들어 단조로움을 깬다(±운빨이 작아 실력이 묻히진 않음).

## 5. 행동 5종 (Actions)

플레이어 장수는 자기 턴에 아래 중 하나를 고른다(적군은 8장 AI가 고름).

### 5.1 공격 (Attack) — 쿨다운 없음
적 하나를 골라 **기본 데미지**:
```
damage = might × 5 × typeMult × moraleMult × variance × (100 / (100 + target.command))
```
- `might`: 공격자 무력
- `typeMult`: 2장 상성 배수
- `moraleMult`: 3.1 사기 배수
- `variance`: rand(0.9, 1.1) — ±10% 변량
- `target.command`: 방어자 통솔 → `100/(100+통솔)`은 통솔 100이면 데미지 절반(×0.5), 통솔 0이면 ×1.0
- **치명타**: 10% 확률로 `× 1.5`, 사기 ±5 (3.2)
- 방어자가 `방어` 중이면 추가로 `× 0.5`
- 최소 데미지 1 (음수/0 방지)

> 〔왜 이렇게?〕 `100/(100+통솔)`은 "방어가 높을수록 덜 아프되, 무한정 무적은 안 됨"(체감 좋은 표준 곡선). 곱셈 구조라 상성·사기·스킬이 **곱으로 누적**돼 역전 한 방이 가능해진다.

### 5.2 방어 (Defend) — 쿨다운 없음
- 다음 자기 턴까지 **받는 데미지 × 0.5**
- 자기 팀 사기 +3

### 5.3 스킬 (Skill) — 쿨다운 3턴, 장수마다 1개
| 스킬 | 효과 |
|---|---|
| `강타` | 단일 대상, 공격 데미지 **×2.0** |
| `연격` | **적 전체**에 각각 공격 데미지 **×0.6** |
| `분기` | 단일 대상 **×1.5** + 자기 팀 사기 **+10** |
| `저격`(궁병) | **체력 가장 낮은 적** 우선 타격 **×1.8** |

(스킬 데미지 = 5.1 공식에 위 배수를 한 번 더 곱함)

### 5.4 책략 (Stratagem) — 쿨다운 3턴, 지력형 장수만
**성공률 공식(공통):**
```
successRate = clamp(50 + (caster.intellect − target.intellect) / 2, 5, 95)   // 단위 %
```
| 책략 | 성공 시 효과 |
|---|---|
| `화계` | 대상에 **화상**: 매 턴 시작 시 대상 maxHp의 **8%** 데미지, **3턴** |
| `교란` | 대상에 **혼란**(2턴) + 적 팀 사기 **−15** |

### 5.5 사기고양 (Rally) — 쿨다운 2턴
- 자기 팀 사기 **+20**. 데미지 없음.

> 〔왜 이렇게?〕 책략·사기고양은 "무력 약한 군사형 장수도 판을 바꾼다"는 레버다. 무력만 높은 가챠 장수가 자동승리하지 못하게 막는 핵심 — 가챠 밸런스의 안전판.

## 6. 상태이상 (Status Effects)

| 상태 | 효과 | 지속 |
|---|---|---|
| `방어` | 받는 데미지 ×0.5 | 1턴 |
| `화상` | 매 턴 maxHp의 8% 데미지 | 3턴 |
| `혼란` | 행동 시 50% 확률로 무작위 대상에게 발동(또는 스킵) | 2턴 |
| `위축` | 공격 데미지 ×0.8, 행동치 −5 (붕괴 중 부여) | 붕괴 동안 |
| `분기` | 공격 데미지 ×1.2, 행동치 +5 (일부 스킬) | 2턴 |

## 7. 라운드 진행 (Game Loop)

```
1. (라운드 시작) 살아있는 장수의 initiative 계산 → 내림차순 정렬
2. 정렬 순서대로 각 장수의 턴:
   a. 턴 시작 처리: 화상 DoT 적용 → 사망 체크
   b. 붕괴 팀이면 40% 동요 → 동요 시 스킵
   c. 혼란 상태면 50% 무작위/스킵 처리
   d. 행동 선택(플레이어 입력 / 적 AI) → 실행 → 데미지/사기/상태 반영
   e. 사망한 장수 처리(처치 시 사기 ±15)
   f. 한쪽 전멸이면 즉시 전투 종료로 점프
3. (라운드 종료) 양 팀 사기 50쪽으로 ±2 수렴 → 붕괴 진입/해제 판정
4. 쿨다운/상태 지속시간 −1
5. 다음 라운드로. 최대 30라운드 도달 시 → 8.2 판정승
```

## 8. 승패 판정 (Victory)

### 8.1 일반 승리
한 팀의 장수 4명이 전부 체력 0 → **상대 팀 승리**.

### 8.2 시간 초과(30라운드) 판정
양쪽 **생존 장수 체력 합(%)** 이 높은 쪽 승리. 동률이면 **사기** 높은 쪽.

### 8.3 결과 등급(margin)
```
승리 팀 생존 장수 수 4명 → "대승"
                  2~3명 → "승"
                  1명   → "신승"(아슬아슬)
패배 측 기준으로 "석패 / 대패"
```

## 9. 결과 출력 (Battle Result) — 바깥 게임으로 넘기는 값

전투 끝나면 아래 객체를 반환한다. **이게 나중에 영지전의 사기·민심 계산 입력이 된다.** (지금은 콘솔/UI에 찍기만)

```json
{
  "winner": "player",
  "rounds": 14,
  "grade": "신승",
  "survivors": [
    { "id": "geunchogo", "hpPct": 41 }
  ],
  "finalMorale": { "player": 62, "enemy": 0 },
  "moraleDiff": 62
}
```

> 〔왜 이렇게?〕 `moraleDiff`·`grade`가 핵심 출력이다. 나중에 "일기토에서 대승 → 적 군 사기 대폭 하락"처럼 영지전에 꽂으면 된다. 지금 이 인터페이스만 맞춰두면 바깥 레이어를 나중에 깔끔히 붙일 수 있다.

## 11. 계산 검증 예시 (Worked Example)

엔진이 맞게 돌아가는지 확인용. 변량은 검증 위해 1.0 고정 가정.

**예시 A — 상성 유리:** 근초고왕(기병, 무력 92)이 적 을지문덕(궁병, 통솔 70) 공격
- 상성: 기병→궁병 = **유리 1.3**
- 사기 50 → moraleMult 1.0
- `damage = 92 × 5 × 1.3 × 1.0 × 1.0 × (100/(100+70))`
- `= 460 × 1.3 × 0.588 = 351`
- 을지문덕 maxHp = 750 × (1+70/200) = 750 × 1.35 = **1012**
- → 약 **3타**에 사망. (명장이 약점 병종을 빠르게 작살내는 그림)

**예시 B — 상성 불리:** 같은 근초고왕(기병)이 적 막리지(창병, 통솔 74) 공격
- 상성: 기병→창병 = **불리 0.8**
- `damage = 92 × 5 × 0.8 × 1.0 × 1.0 × (100/174) = 460 × 0.8 × 0.575 = 212`
- 막리지 maxHp = 1000 × 1.37 = **1370**
- → 약 **6.5타**. 같은 장수라도 상성 앞에선 고전 → "상성으로 역전 가능"

**예시 C — 사기 붕괴 효과:** 적 팀 사기 0(붕괴)일 때 적 광개토대왕의 공격
- moraleMult = 0.8 + 0 = **0.8**, 위축 추가 ×0.8 → 합쳐서 평소의 **64%** 위력
- 게다가 매 턴 40% 확률로 행동 불가 → 대군(고스펙 적)도 무력화

> 위 세 숫자가 코드에서 그대로 나오면 엔진이 정확히 구현된 것.

## 12. 적 AI (간단 규칙 기반, MVP용)

적 장수는 자기 턴에 위에서부터 처음 맞는 규칙을 실행:
1. **이번 턴에 처치 가능한 적**이 있으면 → 그 적을 공격(상성 좋은 아군으로)
2. 자기 팀 사기 < 30 이고 `사기고양` 쿨다운 OK → `사기고양`
3. `책략` 보유 & 쿨다운 OK → 적 **최고 무력** 장수에게 `교란`/`화계`
4. `스킬` 쿨다운 OK → 상성 유리한 적에게 스킬
5. 자기 체력 < 25% → `방어`
6. 그 외 → 체력 가장 낮은 적 중 상성 좋은 대상 공격

> 〔확장 훅〕 나중에 "통솔 = AI 똑똑함" 아이디어를 여기 붙인다. 통솔 높은 지휘관은 위 규칙을 더 많이/정교하게 적용(예: 1·3·4번까지 보고 판단), 낮으면 5·6번 위주의 단순 행동. 카운터스트라이크 봇 난이도처럼. **단, 순수 랜덤 분기는 피한다**(플레이어가 "운빨 탓"으로 느끼지 않게).

## 13. 구현 순서 (Claude Code에게 줄 빌드 순서)

안에서 바깥으로. 각 단계가 끝나면 콘솔에서 숫자로 검증 후 다음으로.

1. **데이터 모델 + 체력/데미지 공식**(1·5.1장) → 11장 예시 A·B 숫자 재현
2. **라운드 루프 + 행동 순서**(4·7장) → 공격만 가능한 4:4가 끝까지 돌아감
3. **나머지 행동**(방어/스킬/사기고양/책략) 추가(5장)
4. **상성 배수** 연결(2장)
5. **팀 사기 + 붕괴**(3장) → 예시 C 검증
6. **상태이상**(6장)
7. **승패 판정 + 결과 출력**(8·9장)
8. **간단 적 AI**(12장)
9. 그 다음에야 화면(UI) — 지형/배경은 단순하게, **장수 초상·일기토 연출에 비주얼 예산 집중**
