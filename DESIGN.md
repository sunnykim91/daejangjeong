## 0. 게임 성격 (중요)

여러 결정이 **서버·계정·실시간 지속 세계**를 전제로 한다:
- 서버 내 시간(오전/오후 순환) · 에너지 자동 충전 · 가챠/과금 인벤토리 · PK 서버 · 군주/군벌 세력.

→ **개발 2트랙으로 분리한다.**
- **Track A (지금·클라이언트만):** 재미의 핵심을 순수 HTML/JS로 검증. 서버 불필요.
  야습·숙영·피로도·혼란·와해, 밤/낮 연출, 병력 표현/정찰, 일기토(턴제), 장비 6슬롯+세트, 지도 통합.
- **Track B (나중·서버 도입):** 온라인·상업층. 계정, 서버시간, 에너지, 가챠 인벤토리, 군벌전, PK/투항, 세계지도 세력.

데이터/로직은 지금도 "순수 모듈"이라 나중에 서버로 옮기기 유리하게 설계되어 있음 — 이 구조를 유지한다.

## A. 방향성

1. **일기토 = 턴제로 복원.** (기획서 명시 · 이전 결정 유지. ※ 친구 최종 확인 권장 — 이번 답변엔 명시 언급 없었음)
2. **지도 전쟁은 하나로 통합.** 최초 프로토타입은 `total.html` 단일 지도 전쟁으로. `campaign.html`(간이형)은 흡수·정리. 핵심: **장군이 군대를 통솔**한다.

## B. 야습 · 혼란 · 와해 (게임 정체성 — 빈번하게)

핵심 루프:
1. **밤(야간)** — 화면이 살짝 어두워짐. 병사들은 **숙영지에서 숙영**.
2. **숙영 중** — 부대 **피로도 빠르게 회복** → 회복될수록 **공격력 상승**. (신규 스탯: 피로도)
3. **야습** — 야간에 적을 기습. **적이 못 알아채고 숙영 모드일 때 성공** → 적군 **혼란 상태** 진입.
4. **와해** — 혼란이 지속되면 와해 → 전투 지속력 상실.
5. **투항 (PK 서버, 아이디어)** — 와해 시 상대 **장수 충성도**에 따라 투항(항복)을 받을 수도.

→ 야습 성공은 "적이 야습을 알아챘는가(정찰/지력/경계)"에 달림. 소수 역전의 핵심 장치.

## C. 시간대 (서버 시간)

- 밤/낮(오전/오후)은 **서버 내 시간**으로 순환. **시간당 두 번** 정도 전환(≈30분 주기).
- 야습 성공률에 영향. → Track A에서는 가짜 시계(클라 벽시계/디버그)로 먼저 구현, Track B에서 서버시간으로 대체.

## C-2. 병력 표현 (⚠️ 미결 — 아이디어 필요)

- 규모: 기병 100명 ~ 10만 명. 화면에 전부 그릴 수 없음.
- 참고: 머그삼국지 = 바둑판 칸의 **색으로 군대 규모** 표시(흰색=10~20만, 분홍=1천 이하 등) → 한눈에 규모 파악.
- 고민: 그러면 **허장성세(블러핑)**가 안 됨. 규모를 감추거나 속일 수 있어야 전략이 산다.
- → 후보 아이디어는 본문 논의 참조. **정찰로 실규모 파악 + 허장성세 방침**으로 블러핑을 시스템화하는 방향 검토.

## 진영 / 장수

- 초기 진영: **위·촉·오 / 고구려·백제·신라 / 일본 전국시대** (동시 시작).
- 확장(나중): 임진왜란(이순신), 초원 세력(징기스칸) 등 시즌.

## D. 상업성 (처음부터 탑재)

- **가챠 = "등용서".** 등급: **신화 / 전설 / 레어 / 노말**.
  - 일일 퀘스트 → 노말 등용서 1장/일.
  - 정벌 콘텐츠(이민족 토벌) → 레어·전설 등용서.
- **과금 모델:**
  - 등용서 세트 (5천원 ~ 10만원).
  - **에너지(행동력):** 출정·전장 진입에 소모. **3~5분당 1개** 자동 충전. 결제로 추가 구매.

## E. 지도 (3단계 — 세계지도 필요)

- **1단계 세계지도 필요.** 고을마다 군주·군벌이 나뉘고, 군주가 나라를 다스리면 **세력을 볼 수 있어야** 함. (군벌전의 전제)
- 2단계 지역/성 지도, 3단계 전장 지형도는 기존 방향 유지.

## 기획자 2차 답변 (2026-07-03 반영)

- **일기토 = 턴제 확정 + RAID(그림자의 전설)식 스킬킷.** 각 장수에 **액티브 스킬 1~4개 + 고유 특성(패시브)**, 이를 조합해 대결. 전쟁은 실시간 RTS 확정. → engine.js 단일스킬을 **스킬킷 시스템으로 확장** 필요(진행 중).
- **개발 순서 동의.** 전쟁 RTS의 전술 활용·유기적 조작을 먼저 완성. 서버는 **서비스 단계에서 오토스케일**로 대처(Track B).
- **병력 표현:** 정찰+허장성세 아이디어 OK지만 확정 아님 — **실제 플레이에서 인식 잘 되는 방식을 채택**(플레이 검증 후 결정).
- **오프라인:** RTS는 전투지에 모인 실시간 대전 → 접속 종료 = **패배 처리**(Track B).
- **투항(장수 충성도): PK 서버 대인전에서만.** AI전엔 없음(어렵게 모은 장수 뺏기면 이탈 위험). → Track A는 투항 없음(전사/도망만).
- **피로도:** 피로 0 = **전투력 100%**(보너스 아님), 과로 시 **패널티(공격력↓·야습 취약)**. → fatigueAtk 상한 1.0로, raidChance에 피로 취약 추가.
- **가챠: 천장 없음.** 최고 네임드(신화 6★: 광개토·관우 등)는 **10~50만원 써야 나올 극악 확률**, 무과금 1~2년에 한둘. → RATES 신화 극악화.
- **진영별 장수 수:** 테스트하며 결정. 보통(노말)·레어 필러 장수 보강(이전·유봉·흑치상지·관창 류). → 노말 풀 확충.
- **에너지:** 플레이타임 3시간 기준(협의 가능).
- **세계지도(Track B):** 군웅할거로 시작 → 군주들이 모여 군벌 형성. **왕 = 성 N개↑ + 옥새 + 천자 유무 + 명성 총합**.
